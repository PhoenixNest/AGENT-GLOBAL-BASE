# Brand Design Department

Responsible for translating product requirements into high-fidelity mobile UI/UX designs and production-grade interactive prototypes. The department owns Stage 2 of the development pipeline and participates as a reviewer in Stages 6, 8, and 10.

> Reports to the Chief Design Officer (CDO) and the Chief Information Officer (CIO).

## Pipeline Stages

| Stage                                   | Role                                                                                                                                                                                         |
| --------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Stage 2** — PRD → Web Prototype + IDS | **Responsible Producer.** CDO analyses PRD + SRD, mobilises team to produce HTML prototypes, convenes CPO/CIO/CTO review, presents to user, produces Interaction Design Specification (IDS). |
| **Stage 6** — Code Review               | **Reviewer.** CDO verifies all design requirements in the IDS and web prototype have been accurately reproduced in the codebase.                                                             |
| **Stage 8** — Integrity Verification    | **Reviewer.** CDO confirms no design specifications were silently reduced or removed during the testing remediation phase.                                                                   |
| **Stage 10** — Release Readiness Check  | **Sign-off authority.** CDO certifies: all CDO/IDS specifications accurately realised.                                                                                                       |

---

## Key Artifacts Produced

- **Web Prototype** — Single HTML file, browser-runnable, no build step required. Validates functional requirements and design style.
- **Interaction Design Specification (IDS)** — Component trees, gesture vocabularies, state diagrams, edge case matrices, and platform-specific interaction patterns (iOS HIG, Android Material Design). Governs native mobile behaviour; the web prototype does not.
