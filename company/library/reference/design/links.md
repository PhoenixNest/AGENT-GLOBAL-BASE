# Design Reference Library

Reference resources for the Brand Design Department and relevant personnel. Use these materials to inform design decisions, maintain platform consistency, and ensure accessibility compliance.

---

## Inspiration & Community

- [Dribbble — Discover the World's Top Designers](https://dribbble.com/)
  Community showcase for mobile UI, product design, and visual inspiration. Browse by category (Mobile, Product Design, UI) or search by topic.

---

## Android Design

- [Material Design 3 — Google's open-source design system](https://m3.material.io/)
  Official design system for Android. Covers components, colour tokens, typography, motion, accessibility, and adaptive layouts. The current standard for Android UI (Material You / Dynamic Colour).

- [Android Design — Figma Community Kit](https://www.figma.com/@androiddesign)
  Official Android Design Figma resources including the Android Design Kit and Material 3 Expressive kits. Structured for accessibility-aware handoff (spacing, contrast, touch target definitions).

- [Android Accessibility Overview](https://developer.android.com/guide/topics/ui/accessibility)
  Semantic modifiers, touch target requirements (48×48dp minimum), TalkBack testing guidance, and Dynamic Type support for Jetpack Compose.

---

## Apple Design

- [Apple Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/)
  Definitive reference for iOS, iPadOS, macOS, watchOS, and visionOS design patterns, components, and system behaviours. Updated alongside new OS releases.

- [Apple Design Resources](https://developer.apple.com/design/resources/)
  Official Figma and Sketch templates, SF Pro / SF Compact / SF Mono typefaces, and SF Symbols library download. Reference-quality submission materials for App Store review.

- [SF Symbols](https://developer.apple.com/sf-symbols/)
  Apple's system iconography library — over 6,000 symbols designed to integrate seamlessly with SF Pro typography. Use for icon consistency across iOS designs.

- [Apple Accessibility — Design Guidance](https://developer.apple.com/accessibility/)
  Perceivable, Operable, Understandable, Robust principles applied to iOS. Covers Dynamic Type, VoiceOver labelling, touch target sizing, contrast requirements, and Reduce Motion support. Use Xcode Accessibility Inspector for real-time auditing.

---

## Interaction & Prototyping

- [Material 3 Component Specs](https://m3.material.io/components)
  Detailed per-component specifications: anatomy, states, behaviour, spacing, and accessibility properties. Reference when producing Interaction Design Specifications (IDS).

- [Apple HIG — Patterns](https://developer.apple.com/design/human-interface-guidelines/patterns)
  Platform-specific interaction patterns: navigation, gestures, feedback, loading, modality. Essential for IDS authorship covering iOS-specific behaviour.
