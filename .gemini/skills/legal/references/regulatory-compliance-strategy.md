# Regulatory Compliance Strategy

| Competency                   | Description                                                                   | Quality Criteria                                                                              |
| ---------------------------- | ----------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| Jurisdiction Assessment      | Produce a jurisdiction-by-jurisdiction compliance matrix for active markets   | Matrix covers: data localisation, cross-border transfer, licensing, and sector-specific rules |
| Transfer Mechanism Selection | Select and implement appropriate cross-border data transfer mechanisms        | Mechanism selected matches current regulatory status; SCCs up to date; no gap in coverage     |
| Compliance Programme Design  | Build a compliance programme covering policy, controls, monitoring, and audit | Programme maps to applicable regulations; control owners assigned; audit cadence defined      |
| Government Relations         | Manage proactive engagement with data protection authorities and regulators   | Engagement strategy documented; no enforcement actions without legal response plan in place   |

## Execution Guidance

### Jurisdiction Compliance Matrix Structure

For each active market, document:

| Field                        | Content                                                              |
| ---------------------------- | -------------------------------------------------------------------- |
| Data localisation rules      | What data must remain in-country; penalty for non-compliance         |
| Cross-border transfer rules  | Permitted mechanisms; adequacy status; SCCs required                 |
| Sector-specific requirements | Transit data, financial data, health data — any special rules        |
| Licensing requirements       | Operating licences; data processing registrations                    |
| Enforcement authority        | Name of data protection authority; contact; enforcement track record |
| Current compliance status    | Green / Amber / Red with outstanding action items                    |

### Cross-Border Transfer Mechanism Selection

| Scenario                              | Recommended Mechanism                                                      |
| ------------------------------------- | -------------------------------------------------------------------------- |
| EU → adequate country                 | Adequacy decision (no SCCs required)                                       |
| EU → non-adequate country             | Standard Contractual Clauses (2021 version); supplement with TIA           |
| Intra-group transfers (multinational) | Binding Corporate Rules (BCRs) where group-wide; SCCs for project-specific |
| Singapore PDPA transfers              | PDPA contractual clauses or adequacy whitelist                             |
| Nigeria NDPR transfers                | NDPR consent + contractual safeguards; register with NITDA if applicable   |
| APAC multi-country (non-EU)           | APEC CBPR Framework where available; bilateral contractual safeguards      |
