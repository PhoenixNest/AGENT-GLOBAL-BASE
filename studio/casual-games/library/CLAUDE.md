# studio/casual-games/library/ — Studio Knowledge Hub

Central knowledge repository for the Casual Games Studio. Start here for studio orientation,
domain topics, and strategic context before working on any studio task.

---

## Directory Structure

```
library/
├── overview/
│   └── casual-games-studio.md    ← Studio charter, strategic brief, vision, crew summary
├── topics/                       ← Domain-specific knowledge
└── references/                   ← External references and resources
```

---

## Navigation Guide

| I need…                                      | Go to                                  |
| -------------------------------------------- | -------------------------------------- |
| Studio charter, vision, strategic brief      | `overview/casual-games-studio.md`      |
| Domain-specific knowledge (audio, art, etc.) | `topics/`                              |
| External references                          | `references/`                          |
| The 11-stage pipeline                        | `../pipeline/casual-games-pipeline.md` |
| Crew structure and division roster           | `../team/README.md`                    |
| Active or planned game projects              | `../projects/`                         |

**Read `overview/casual-games-studio.md` first** for any new studio task — it is the authoritative
strategic brief and provides the clearest orientation to what the studio is and where it is going.

---

## Document Authority

When library content conflicts with another source:

1. `../pipeline/casual-games-pipeline.md` — canonical truth for the pipeline
2. `../team/crew/<division>/<role>/<name>/agent/profile.md` — canonical crew agent identity
3. `library/overview/casual-games-studio.md` — authoritative studio summary
4. `library/topics/` — authoritative for domain knowledge

Library documents may lag the canonical pipeline or crew profiles. When they differ, the canonical
source wins.

---

## Rules

- Do not produce studio deliverables without first reading the strategic brief in `overview/`.
- Topic files are authoritative for their domain within the studio context — treat them as
  executable guidance, not suggestions.
- This library is specific to the Casual Games Studio. Do not mix content from the company library
  (`company/library/`) into studio documents.
