# Game Design Document Patterns

**Applies To:** All game design documentation in the Casual Games Studio

---

## GDD Required Sections

1. **Game Overview** — Title, genre, platform, target audience, USPs
2. **Game Concept** — High concept, core gameplay loop, game pillars
3. **Gameplay Mechanics** — Core/secondary mechanics, controls, difficulty curve
4. **Game Systems** — Progression, economy, reward, social features
5. **Content Structure** — Levels, characters, items, environments
6. **Monetization Design** — Business model, IAP categories, conversion funnels
7. **Meta Game** — Long-term goals, events, collections, achievements
8. **User Experience** — Onboarding, UI/UX flow, feedback systems, accessibility
9. **Technical Considerations** — Performance targets, platform requirements, save system
10. **Competitive Analysis** — Competitor games, market positioning

---

## GDS Required Sections

Expands on the GDD with: detailed mechanics formulas + state machines, content specs (level layouts, stat sheets, dialogue), UI/UX wireframes, audio specs, art specs.

---

## Core Loop Design Principles

- Completable in 30–60 seconds
- Should be satisfying and rewarding
- Has clear goals and feedback
- Scales in complexity over time

## F2P Best Practices

- Value proposition: players feel they're getting value
- Non-intrusive: never interrupt core gameplay
- Optional: never require payment to progress
- Fair: avoid pay-to-win mechanics
