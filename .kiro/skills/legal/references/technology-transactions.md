# Technology Transactions

| Competency             | Description                                                                      | Quality Criteria                                                                                    |
| ---------------------- | -------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| Contract Negotiation   | Lead negotiation strategy for technology agreements, SaaS, and API licences      | All material risk positions documented pre-negotiation; fallback positions pre-approved by CLO      |

## Execution Guidance

### Contract Tier Model

| Tier   | Value Range  | Template                     | Approval Required      | Target Cycle Time  |
| ------ | ------------ | ---------------------------- | ---------------------- | ------------------ |
| Tier 3 | < $50K       | Standard MSA + SOW           | Senior Counsel         | ≤ 3 business days  |
| Tier 2 | $50K – $500K | Enhanced MSA + SOW           | CLO review             | ≤ 7 business days  |
| Tier 1 | > $500K      | Fully negotiated agreement   | CLO sign-off + CTO/CPO | ≤ 15 business days |
| Gov    | Any value    | Government contract playbook | CLO + User sign-off    | As required        |

### Government Contract Mandatory Provisions

Every government technology agreement must include:

1. **Data sovereignty clause** — Specifying where data is processed and stored; cross-border transfer restrictions
2. **Audit rights** — Client right to audit compliance with contractual obligations
3. **Security certification requirements** — Applicable standards (ISO 27001, SOC 2, OWASP MASVS) and attestation cadence
4. **Termination for convenience** — Compensation framework for early termination
5. **IP ownership** — Clear allocation of background IP, developed IP, and client data IP
6. **Escrow provisions** — For mission-critical software where continuity is required
